Why it matters

Why do we use Groupings?

Raw data is one row per event — a row per request, per invoice, per check. That's thousands of rows. Grouping collapses them into something you can actually read and act on.

Without grouping

You see 50,000 individual check rows — impossible to tell which company is your biggest customer or which month was best.

With grouping by Company

One row per customer showing total revenue, request count, and average order value — instantly comparable.

With grouping by Month

One row per month showing volume and revenue — see the trend, spot the dip, understand growth.

Common use cases

Revenue per account Requests by check type Monthly volume trend Top 10 customers Checks by region Renewals by AM
Tutorial 1

How to Group a Table

Your goal: collapse 50,000 request rows into one row per company with a total revenue column.

1

Open a workbook with a data table (e.g. a requests or revenue table)

2

Click on the column you want to group by — e.g. Company Name

3

In the dropdown that appears, select Group Column

4

The table collapses — you now see one row per unique company instead of one per request

5

Click + at the top right of the table → Add Column

6

In the formula bar, type Sum([Revenue]) and press Enter

7

Double-click the column header to rename it "Total Revenue" — you now have revenue per company

Functions

Aggregate Functions

Once your table is grouped, aggregate functions calculate across all rows in each group — returning one answer per group.

The Basics

  • Sum([Revenue]) — total
  • Avg([Revenue]) — average
  • Count([ID]) — row count
  • CountDistinct([ID]) — unique count, no duplicates
  • Min([Col]) / Max([Col])

Conditional — only when true

  • SumIf([Amount], [Status]="Paid") — sum only paid rows
  • CountIf([Col], [Region]="North") — count matching rows
  • AvgIf / MaxIf / MinIf — same pattern

Totals & Share

  • GrandTotal(Sum([Revenue])) — total across ALL groups
  • PercentOfTotal(Sum([Revenue])) — this group's share of total
  • Subtotal([Revenue]) — sub-group total

Functions

Key Functions: Text & Contains

Contains — does this text include a word?

Returns True or False. Case-sensitive — always wrap with Lower() to be safe.

Contains([Company Name], "Ltd")→True or False
Contains(Lower([Product]), "camera")→Case-insensitive match
Contains([Notes], "urgent", "ASAP")→True if either word appears

Type Conversion

  • Text([Col]) — convert to text
  • Number([Col]) — convert to number
  • Date([Col]) — convert to date
  • Logical([Col]) — convert to True/False

Text Functions

  • Concat([A], " ", [B]) — join strings together
  • Lower() / Upper() — change case
  • Proper([Col]) — Title Case
  • Trim([Col]) — remove leading/trailing spaces
  • Left([Col], 3) / Right([Col], 3) — extract characters
  • Len([Col]) — character count
  • Replace([Col], "old", "new") — substitute text
  • SplitPart([Col], "-", 1) — split by delimiter
Functions

Window Functions: What & Why

Aggregates collapse rows down to one per group. Window functions keep every row but add a new calculated value based on surrounding rows — great for trends, rankings, and comparisons.

Running Total

"How much revenue have we made so far this year?" — grows with each month

Ranking

"Which company is our #1 account?" — ranks every row automatically

Month-on-Month

"Did we grow vs last month?" — compares each row to the one above it

Rolling Average

"What's our 3-month average?" — smooths out noisy week-to-week data

Functions

Window Functions: Examples

Running Total — revenue so far this year

CumulativeSum(Sum([Revenue]))

Jan: £10k → Feb: £22k → Mar: £38k — grows each month

Ranking — who's our top account?

Rank(Sum([Revenue]))

Every company gets a position number: 1st, 2nd, 3rd. Ties handled automatically.

Split into Quartiles — top/bottom 25%

Ntile(Sum([Revenue]), 4)

Assigns 1–4 to every account. 1 = bottom quarter, 4 = top quarter.

Month-on-Month — did we grow?

Sum([Revenue]) - Lag(Sum([Revenue]), 1)

Lag(..., 1) pulls the value from the row above — i.e. the previous month. Subtract to get the change.

Rolling 3-Month Average — smooth the noise

MovingAvg(Sum([Revenue]), 3)

Averages the current month and 2 surrounding months — flattens spikes so trends are clearer.

Fill in Blanks — no more null gaps

FillDown([Account Manager])

If a column has nulls, carries the last known value down.

More PM Features

Pivot Tables

Compare across two dimensions at once. Revenue by Company (rows) × Month (columns) — every cell is that company's revenue in that month.

Lookups / Joins

Pull in data from a second table using a shared ID — like VLOOKUP. Add AM owner, contract dates, or tier to any requests table.

Conditional Formatting

Charts

Any grouped table becomes a bar, line, or scatter chart in one click. Change the data, the chart updates automatically.

Filter any column directly, or use page-level controls (date pickers, dropdowns) that update every element on the page at once.

Export to CSV

Three-dot menu on any table → Export → Download as CSV. Your active filters apply — you get exactly what you see.

Combining Functions

You can nest functions inside each other. Start simple — write the inner function first, verify it works, then wrap it.

Contains(Lower([Company Name]), "ltd")→Case-insensitive flag
SumIf([Revenue], Contains(Lower([Category]), "software"))→Revenue from software rows only
Sum([Revenue]) / GrandTotal(Sum([Revenue]))→This group's % of total
Sum([Revenue]) - Lag(Sum([Revenue]), 1)→Growth vs prior period
If(Sum([Revenue]) > 10000, "High", If(Sum([Revenue]) > 5000, "Mid", "Low"))→Tier label

Key Principles

Things to Remember

You can't break anything

You're in Explore mode by default. All your changes are personal and don't affect what others see. Experiment freely.

Aggregate functions need a grouped table

Sum(), Count(), Avg() only make sense once you've grouped — otherwise they return one number for the whole table.

Window functions keep all rows; aggregates collapse them

Use aggregates when you want one result per group. Use window functions when you want every row to keep its own value plus a calculated insight.

Functions are case-sensitive — always use Lower()

"Ltd", "LTD", and "ltd" are treated as different strings. Wrap with Lower() whenever matching text.

GrandTotal() ignores your current grouping

Useful for % of total calculations — it reaches outside the current group to get the true overall total.
